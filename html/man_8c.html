Detailed Description

Company: Hunting Titan

File Name: man.c

Summary: These files receive and process the Manchester encoded data from the PerfSwitch.

Description: These files receive and process the Manchester encoded data from the PerfSwitch. Three state machines are used, MAN_tasks, MAN_RX_Tasks and MAN_Process_tasks. MAN_Tasks controls the operation of the other two machines. MAN_RX_Tasks begins after an FSK message is sent to a PerfSwitch. MAN_Process_Tasks begins after the MAN_RX_Tasks have completed. MAN_RX_Tasks is called from the input capture function in system_interrupt.c. If there is an error in the PerfSwtich response, the FSK frequencies are adjusted and the message is resent. Three attempts are made for a given message. If no proper response is received from the PerfSwitch, a status bit is set to indicate that we did not receive a response. The states are configured so that other system tasks can be maintained while these tasks are waiting for an event to occur.

◆ MAN_RX_Tasks()

void MAN_RX_Tasks ( void )

Function: void MAN_RX_Tasks(void)

Summary: MAN_Rx tasks function

Description: This routine is the MAN_Rx tasks function. It defines the MAN_Rx's state machine and core logic. It determines if a proper message preamble has been received and determines the time and input level state when a transition occurs. This routine is called when there is a high to low change on the port pin.

The state MAN_RX_STATE_IDLE has no effect on Manchester processing. The state MAN_RX_STATE_PREAMBLE is responsible for syncing up to a 1111. The state MAN_RX_STATE_DATA receives the data. The time ranges allow for variances between PerfSwitches and how they perform over temperature.

Precondition: The system ("SYS_Initialize") and application ("APP_Initialize")initialization should be called before calling this.

Parameters: None.

Returns: None.

Example: MAN_RX_Tasks()

Remarks: This routine must be called from the MAN_Tasks() routine.
