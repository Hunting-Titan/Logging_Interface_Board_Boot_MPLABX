Function: static float32_t Calc_Fsk_Scaling( void )

Summary: Function to calculate the amount of FSK adjustment needed based on the preamble bit timer counts.

Description: Function to calculate the amount of FSK adjustment needed based on the preamble bit timer counts. The average of the timer counts is used to determine the amount of adjustment. The result is a multiplier used in the Generate_Sine_Wave _Data function to change the amount of time between the FSK DAC updates. Adjustments up to 10% have been seen in temperature testing. Adjustments are generally greater than 1 when a PerfSwitch is hot and less than 1 when a PerfSwitch is cold.

◆ MAN_PROCESS_Tasks()

static void MAN_PROCESS_Tasks ( void )
static

Function: static void MAN_PROCESS_Tasks(void)

Summary: MAN_PROCESS tasks function

Description: This routine is the MAN_PROCESS tasks function. It defines the MAN_PROCESS's state machine and core logic. This machine controls the MAN_Process operation. It decodes, packages, tests and prepares for another message to process.

During packaging, the 32 bit Manchester response is packaged into a 4 byte array.

During testing, the CRC is checked and the appropriate response is prepared to send to the host. If a bad response was received, the FSK frequencies are adjusted and two more attempts are made to retrieve a valid response.

Precondition: The system ("SYS_Initialize") and application ("APP_Initialize")initialization should be called before calling this.

Parameters: None.

Returns: None.

Example: MAN_PROCESS_Tasks()

Remarks: This routine must be called from the MAN_Tasks() routine.

◆ MAN_RX_Tasks()

void MAN_RX_Tasks ( void )

Function: void MAN_RX_Tasks(void)

Summary: MAN_Rx tasks function

Description: This routine is the MAN_Rx tasks function. It defines the MAN_Rx's state machine and core logic. It determines if a proper message preamble has been received and determines the time and input level state when a transition occurs. This routine is called when there is a high to low change on the port pin.

The state MAN_RX_STATE_IDLE has no effect on Manchester processing. The state MAN_RX_STATE_PREAMBLE is responsible for syncing up to a 1111. The state MAN_RX_STATE_DATA receives the data. The time ranges allow for variances between PerfSwitches and how they perform over temperature.

Precondition: The system ("SYS_Initialize") and application ("APP_Initialize")initialization should be called before calling this.

Parameters: None.

Returns: None.

Example: MAN_RX_Tasks()

Remarks: This routine must be called from the MAN_Tasks() routine.

Function: static void Test_Manchester (void)

Summary: Function to test the returned Manchester message.

Description: Function to test the returned Manchester message. The first two bytes are checked to make sure they are not both 0. If they are not, the FSK frequencies are adjusted, and the CRC is checked. If the CRC is a match, then a response message to the host is prepared. Otherwise, up to two different FSK frequencies are tried in order to establish communications with a PerfSwitch.
